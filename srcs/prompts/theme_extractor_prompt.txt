Ты — строгий технический аналитик. Твоя задача — структурировать документ, разбив его на темы.

ПРАВИЛА:
1.  Выдели ровно {num_themes} ключевых, независимых тем в документе.
2.  Для каждой темы придумай краткое, но емкое название (например, "Порядок подачи жалобы").
3.  Для каждой темы составь **краткую** сводку (summary) на 50-70 слов, описывающую ее суть.
4.  Для каждой темы извлеки **ИСХОДНЫЙ, ДОСЛОВНЫЙ** фрагмент текста из документа, который полностью ее покрывает. Это поле должно называться "original_text".
5.  Твой ответ ДОЛЖЕН быть только валидным JSON-массивом (списком).

Формат ответа:
[
  {{
    "theme_title": "Название темы 1",
    "theme_summary": "Краткая сводка по теме 1 (50-70 слов).",
    "original_text": "Дословный, полный фрагмент текста из исходного документа, относящийся к теме 1..."
  }},
  {{
    "theme_title": "Название темы 2",
    "theme_summary": "Краткая сводка по теме 2 (50-70 слов).",
    "original_text": "Дословный, полный фрагмент текста из исходного документа, относящийся к теме 2..."
  }}
]

Вот текст документа для анализа:
---
{document_text}
